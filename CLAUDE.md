# CLAUDE.md

## Project Philosophy

This is a minimal Android development environment that avoids Android Studio entirely. Everything runs from the command line and a text editor. The setup prioritizes:

- **No IDE.** No Android Studio. No GUI tools. Just a terminal and an editor.
- **No system pollution.** All dependencies live in the Nix shell or the local `.gradle/` directory. Nothing is installed system-wide except Nix itself.
- **Simplicity.** The project uses plain Java (not Kotlin), XML layouts, and Gradle for builds. No Jetpack Compose, no complex abstractions.
- **C-programmer mindset.** Edit, compile, install, debug with printf (`Log.d`). The Makefile is the entry point for all build operations.

## Environment

You are running inside a **Nix shell** (`nix-shell`). The shell provides:

- **JDK 17** — Java compiler and runtime
- **Gradle** — build system
- **Android SDK** — platform-tools, build-tools 34.0.0, platform API 34

Key environment variables (set by `shell.nix`):

- `$ANDROID_HOME` / `$ANDROID_SDK_ROOT` — points to the Nix-managed Android SDK
- `$GRADLE_USER_HOME` — set to `.gradle/` (local to the project, not `~/.gradle`)

The file `local.properties` is auto-generated by the shell hook with the correct `sdk.dir`. Do not edit it manually or commit it to git.

## Build Commands

Always use the Makefile:

| Command        | What it does                              |
|----------------|-------------------------------------------|
| `make`         | Build debug APK                           |
| `make install` | Build and install on connected device     |
| `make run`     | Build, install, and launch the app        |
| `make release` | Build release APK                         |
| `make clean`   | Remove all build artifacts                |
| `make logcat`  | Stream filtered logs from the device      |

The debug APK is output to `app/build/outputs/apk/debug/app-debug.apk`.

## Coding Guidelines

- **Java only.** Do not introduce Kotlin or Jetpack Compose.
- **Use `android.app.*` and `android.widget.*` directly.** Avoid AndroidX unless explicitly requested. The project currently has no AndroidX dependency.
- **XML layouts.** Use traditional XML layout files in `res/layout/`. No programmatic UI unless there's a good reason.
- **Keep it minimal.** Do not add dependencies unless asked. When adding dependencies, put them in `app/build.gradle` under `dependencies {}`.
- **Logging.** Use `android.util.Log` with a consistent tag. Example: `Log.d("MyApp", "message")`.
- **No code generation tools or annotation processors** unless explicitly requested.

## Adding Dependencies

Edit `app/build.gradle`:

```groovy
dependencies {
    implementation 'com.google.code.gson:gson:2.11.0'
}
```

Then `make` — Gradle handles the download, classpath, and DEX merging.

## Adding a New Activity

1. Create the Java file under `app/src/main/java/com/example/myapp/`
2. Create a layout XML under `app/src/main/res/layout/`
3. Register it in `app/src/main/AndroidManifest.xml`

## Debugging

```bash
# All logs from the app
make logcat

# All errors
adb logcat *:E

# Interactive log filtering
adb logcat | grep "MyApp"

# Check if device is connected
adb devices
```

## Things to Avoid

- Do not suggest or use Android Studio, IntelliJ, or any IDE.
- Do not run `gradle` directly — use the Makefile targets.
- Do not modify `local.properties` — it is auto-generated.
- Do not add Gradle wrapper files (`gradlew`, `gradle/wrapper/`). Gradle is provided by Nix.
- Do not install anything globally on the system. Everything goes through Nix or Gradle.
- Do not create files outside the project directory.
